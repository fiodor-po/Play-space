# Task Brief Template

Use this template to prepare narrow executor-ready implementation work.

## Task
State exactly what should change.

## Goal
Describe the product or technical outcome that must be true after the pass.

## Constraints
List what must not change and what is explicitly out of scope.

## Relevant context
Include the minimum background needed to explain why this task exists now.

## Files to inspect first
List the files or modules that should be read before editing.

## Deliverables
State what the executor must return.

## Validation
List the required build, test, or manual QA checks.

## Stop conditions
State when the executor should stop instead of forcing a broad or risky change.
